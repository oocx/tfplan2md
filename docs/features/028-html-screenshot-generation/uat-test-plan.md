# UAT Test Plan: HTML Screenshot Generation Tool

## Goal
Verify that the Oocx.TfPlan2Md.ScreenshotGenerator tool correctly generates screenshots from HTML reports in various formats and sizes, and that it integrates well with the HTML renderer.

## Artifacts
**Artifact to use:** [artifacts/comprehensive-demo.github.png](artifacts/comprehensive-demo.github.png)

**Creation Instructions:**
1. Ensure [artifacts/comprehensive-demo.github.html](artifacts/comprehensive-demo.github.html) exists (generated by feature 027).
2. Run the screenshot tool:
   ```bash
   dotnet run --project src/tools/Oocx.TfPlan2Md.ScreenshotGenerator -- \
     --input artifacts/comprehensive-demo.github.html \
     --output artifacts/comprehensive-demo.github.png \
     --full-page
   ```

## Test Steps
1. Run the command above to generate the screenshot.
2. Open [artifacts/comprehensive-demo.github.png](artifacts/comprehensive-demo.github.png) and inspect it.
3. Run the tool with a custom viewport:
   ```bash
   dotnet run --project src/tools/Oocx.TfPlan2Md.ScreenshotGenerator -- \
     --input artifacts/comprehensive-demo.github.html \
     --output artifacts/comprehensive-demo-mobile.png \
     --width 375 --height 667
   ```
4. Inspect [artifacts/comprehensive-demo-mobile.png](artifacts/comprehensive-demo-mobile.png).

## Validation Instructions (Test Description)

**Specific Resources/Sections:**
- The entire report in [artifacts/comprehensive-demo.github.png](artifacts/comprehensive-demo.github.png).
- The mobile layout in [artifacts/comprehensive-demo-mobile.png](artifacts/comprehensive-demo-mobile.png).

**Exact Attributes:**
- Check that the full-page screenshot contains all resources from the plan (it should be a very tall image).
- Check that the mobile screenshot (375px wide) shows the mobile-optimized layout (e.g., stacked tables or hidden columns if implemented in the HTML template).

**Expected Outcome:**
- [artifacts/comprehensive-demo.github.png](artifacts/comprehensive-demo.github.png) should be a high-quality, full-length image of the report.
- [artifacts/comprehensive-demo-mobile.png](artifacts/comprehensive-demo-mobile.png) should be exactly 375x667 pixels and show the report rendered for a small screen.
- No text should be cut off horizontally in the mobile view.

**Before/After Context:**
- Previously, we only had HTML reports which required a browser to view. Now we can generate static images for easy sharing in PR comments or documentation.
