tfplan2md

Convert Terraform plan JSON files into human-readable Markdown reports optimized for GitHub and Azure DevOps pull request comments.

Note: This project was developed 100% with GitHub Copilot to explore AI-assisted development capabilities.

Overview

Terraform plans are notoriously difficult to review in pull requests. Raw terraform plan output is verbose and cryptic. terraform show -json provides complete data but is unreadable for humans.

The Problem

  • Wall of text output ‚Äî Raw plan output is verbose and hard to scan
  • No structure ‚Äî Changes aren't grouped logically
  • Index-based diffs ‚Äî List changes show as confusing modifications
  • Lost context ‚Äî Hard to see what's actually changing

The Solution

tfplan2md generates clean, readable Markdown reports that structure changes logically, show semantic diffs, and render beautifully in PR comments.

Quickstart

Prerequisites

  • Docker installed
  • Terraform plan file (JSON format)

Step 1: Pull the image

docker pull oocx/tfplan2md:latest

Step 2: Run with your plan

terraform show -json plan.tfplan | docker run -i oocx/tfplan2md

Step 3: Integrate with CI/CD

- name: Generate plan report
  run: |
    terraform show -json plan.tfplan | \
      docker run -i oocx/tfplan2md > plan.md
    
- name: Comment on PR
  uses: actions/github-script@v6
  with:
    script: |
      const fs = require('fs');
      const report = fs.readFileSync('plan.md', 'utf8');
      github.rest.issues.createComment({
        issue_number: context.issue.number,
        owner: context.repo.owner,
        repo: context.repo.repo,
        body: report
      });
