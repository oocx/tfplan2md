Generate clean, structured reports from Terraform plan JSON that render beautifully in GitHub and Azure DevOps PR comments. No more walls of text.

The Problem

Terraform plans in pull requests are painful to review:

  • ‚ùå Verbose output ‚Äî Raw plan output is overwhelming
  • ‚ùå No structure ‚Äî Changes aren't logically grouped
  • ‚ùå Cryptic diffs ‚Äî Index changes instead of semantic diffs
  • ‚ùå Lost context ‚Äî Can't see the big picture

Quick Start

1

Pull the Docker image

bash 
docker pull oocx/tfplan2md:latest
2

Pipe your Terraform plan

bash 
terraform show -json plan.tfplan | docker run -i oocx/tfplan2md

The report is written to stdout. Redirect to a file or pipe to your PR comment tool.

3

Integrate with CI/CD

yaml 
- name: Generate markdown report
  run: terraform show -json plan.tfplan | docker run -i oocx/tfplan2md > plan.md

- name: Post PR comment
  uses: actions/github-script@v6
  with:
    script: |
      const fs = require('fs');
      const body = fs.readFileSync('plan.md', 'utf8');
      await github.rest.issues.createComment({
        issue_number: context.issue.number,
        owner: context.repo.owner,
        repo: context.repo.repo,
        body
      });
