Sensitive Value Masking

Sensitive values are automatically masked to prevent accidental exposure in pull requests.

all providers

Terraform marks certain values as sensitive. tfplan2md respects these markings and ensures they are never displayed in the generated markdown report, replacing them with (sensitive value).

Raw Terraform Output 
# module.security.azurerm_key_vault_secret.db_password will be created
+ resource "azurerm_key_vault_secret" "db_password" {
    + content_type = "password"
    + id           = (known after apply)
    + key_vault_id = "/subscriptions/12345678-1234-1234-1234-123456789012/resourceGroups/rg-tfplan2md-demo/providers/Microsoft.KeyVault/vaults/kv-tfplan2md"
    + name         = "db-password"
    ~ value = (sensitive value)
    # At least one attribute in this block is (or was) sensitive,
    # so its contents will not be displayed.
  }

Security First

  • Automatically detects sensitive values from the plan JSON
  • Masks values in tables, diffs, and summaries
  • Prevents accidental leakage of secrets in PR comments
  • Works for all providers and resources

Configuration

Use the --show-sensitive flag to reveal sensitive values when needed:

Command 
# Default: sensitive values masked
tfplan2md plan.json

# Show sensitive values (use with caution!)
tfplan2md plan.json --show-sensitive

Learn More

Sensitive value masking is enabled by default and respects Terraform's sensitive attribute markers.
